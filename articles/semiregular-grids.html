k-Uniform Tilings

Now let's start working on implementing an applet that could render any semi-regular tiling. The approach is as follows:

  • Pick a point, this will be the initial vertex.
  • Surround that vertex by the polygons contained in the pattern
  • For each new vertex, consider whether there is only one way to fill in the remaining gaps with polygons such that the pattern would be preserved. If so, add such polygons, which will inevitably produce new vertices.
  • Rinse and repeat until the viewport is filled

There are some complications to consider. For some tilings, such as the '4.6.12', you also need to consider the mirrored version of the tiling, i.e. '6.4.12', otherwise the process will stop very early. For other tilings, such as the '3.3.3.3.6' tiling, there is an inherent ambiguity that requires a choice to be made.

In fact, if we want to extend this to k-vertex tilings, we will very quickly run into ambiguous cases.
